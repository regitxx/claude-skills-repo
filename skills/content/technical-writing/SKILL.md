<when_to_use>
Use this skill when:
- Writing API documentation
- Creating how-to guides and tutorials
- Documenting system architecture
- Writing technical proposals or RFCs
- Creating installation and setup guides
- Writing troubleshooting documentation
</when_to_use>

<instructions>
1. **Understand Your Audience**
   - Technical level (beginner, intermediate, expert)
   - Background (frontend, backend, DevOps, etc.)
   - Goals (learn, reference, implement, debug)
   - Environment (local, cloud, CI/CD, etc.)

2. **Structure for Clarity**
   - Lead with purpose and outcome
   - Use clear headings and hierarchy
   - Break complex ideas into sections
   - Use consistent terminology
   - Provide examples early

3. **Write Clear Instructions**
   - Use imperative voice (do this)
   - Numbered steps for procedures
   - Include expected outcomes
   - Provide error handling guidance
   - Link to related topics

4. **Include Practical Examples**
   - Code snippets with context
   - Expected input and output
   - Common variations and use cases
   - Error scenarios and solutions
   - Real-world application

5. **Format for Readability**
   - Use code blocks with syntax highlighting
   - Employ tables for comparisons
   - Add diagrams for complex concepts
   - Highlight important information
   - Keep paragraphs short (2-3 sentences)
</instructions>

<guidelines>
**Do:**
- Use active voice and imperative mood
- Define acronyms on first use
- Provide copy-paste ready examples
- Include expected output/results
- Add troubleshooting sections
- Update examples when API changes
- Link between related topics
- Test examples before publishing

**Don't:**
- Use "I" or "we" (maintain neutrality)
- Include unnecessary marketing language
- Assume reader knows domain knowledge
- Leave examples incomplete or broken
- Hide important warnings in prose
- Make paragraphs longer than 3 sentences

**Formatting Standards:**
- Headings: H2 for sections, H3 for subsections
- Code blocks: Always specify language
- Commands: Use code formatting
- Important info: Use blockquotes or callouts
</guidelines>
